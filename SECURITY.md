# Security Policy

## Supported Versions

Security fixes are applied to the latest release on the `main` branch. Older
versions are not actively supported unless noted in a release.

| Version | Supported          |
| ------- | ------------------ |
| latest on `main` | :white_check_mark: |
| older tags       | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly rather
than opening a public issue.

**Preferred:** use [GitHub private vulnerability reporting](https://github.com/markheydon/meaty-times/security/advisories/new) for this repository.

**Alternative:** contact [@markheydon](https://github.com/markheydon) via GitHub with details of the issue.

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept if available
- Affected versions or branches, if known

## What to Expect

- **Acknowledgement** within a reasonable timeframe (typically a few business days)
- **Assessment** of severity and affected scope
- **Fix or mitigation** on `main` where appropriate, with credit given unless you prefer to remain anonymous
- **Coordinated disclosure** — please allow time to address the issue before public disclosure

## Out of Scope

The following are generally out of scope for this project:

- Social engineering or physical attacks
- Denial-of-service attacks against public deployments you do not own
- Issues in third-party dependencies already fixed in a newer supported version (report those upstream, or open a Dependabot-driven update PR here)
- Cooking-time accuracy or food-safety guidance disputes (see [contributing](CONTRIBUTING.md) for rule changes)

Thank you for helping keep MeatyTimes and its users safe.
